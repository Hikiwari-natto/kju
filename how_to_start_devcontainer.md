# PyCharmでDevContainerを起動する手順

PyCharmでDevContainer（開発コンテナ）を起動する手順を、初心者の方にもわかりやすく解説します。

### 1. プロジェクトをPyCharmで開く
まず、このプロジェクト（`DevCont TDL Website`）のフォルダをPyCharmで開いてください。

### 2. コンテナでの起動（自動認識）
フォルダを開くと、画面の右下に「**Dev Container detected**（開発コンテナが検出されました）」という通知が表示されることがあります。

*   その通知にある **「Reopen in Container」** という青い文字をクリックしてください。
*   これだけで、PyCharmが自動的にコンテナの準備（ビルド）を開始し、その中でプロジェクトを再度開いてくれます。

### 3. 通知が出ない場合（手動で起動）
もし通知が消えてしまったり表示されなかったりした場合は、以下の手順で起動できます。

1.  **左側のプロジェクトツリー**から `.devcontainer` フォルダの中にある `devcontainer.json` を右クリックします。
2.  メニューの中にある **「Create Dev Container」**（または Dev Container 関連の項目）を選択します。
3.  あるいは、PyCharmの画面の**一番左下にある緑色のアイコン**（ステータスバーの部分）をクリックして、「**Reopen in Container**」を選択してください。

### 4. 準備が完了するのを待つ
初めて起動するときは、OSやPythonの「イメージ」をダウンロードして環境を作るため、数分かかることがあります。
*   画面右下に進行状況が表示されるので、終わるまで待ちます。
*   完了すると、PyCharmのウィンドウタイトルに `[Dev Container: Python 3]` のような表示が追加されます。

### 5. 動作確認
以前設定した通り、コンテナ内の `/usr/local/bin/python3` が自動的に「インタープリター（通訳者）」として設定されます。
*   PyCharm内のターミナルを開いたとき、自動的にコンテナの中の環境につながっていれば成功です！

### うまくいかないときは？
*   **Docker Desktop** がインストールされていて、起動しているか確認してください（コンテナを動かすには Docker が必要です）。
*   PyCharm のバージョンが **Professional 版** である必要があります（Community 版では標準で DevContainer 機能がサポートされていない場合があります）。

これで、誰でも同じ Python 3.11 の環境で開発を始めることができます。
